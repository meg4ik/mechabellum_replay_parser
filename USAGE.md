# Mechabellum AI Coach — Руководство пользователя

## Что это такое

AI-тренер, который смотрит твои реплеи в реальном времени и даёт тактические советы между раундами:
что купить, куда расставить юниты и как ответить на угрозы противника.

Работает так: Steam сохраняет файл реплея → тренер его читает → просит ввести supply → думает несколько секунд → показывает совет прямо на экране.

---

## Схема работы

```
[Mechabellum] → Steam сохраняет .grbr
                         │
                    [Docker backend]
                         │
              ┌──────────┴──────────┐
              │  Парсинг реплея     │
              │  AI анализ позиции  │
              │  База данных        │
              └──────────┬──────────┘
                         │
                  [Нативное окно на хосте]
                  ┌──────────────────┐
                  │ Ввод supply      │
                  │ → Загрузка...    │
                  │ → Совет + доска  │
                  └──────────────────┘
```

Docker-бэкенд — это мозг (парсинг, AI, БД).
Tkinter-окно — это интерфейс, оно запускается прямо на твоём компьютере.

## Установка uv на Windows

`uv` — это менеджер пакетов для Python. Если при вводе `uv` в терминале появляется ошибка **"не является внутренней или внешней командой"**, нужно установить его правильно.

**Способ 1 — через PowerShell (рекомендуется):**

Открой **PowerShell** (не CMD) и выполни:
```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

После этого **закрой терминал и открой новый** — это важно, чтобы PATH обновился.

**Способ 2 — если уже установлен через pip, но не в PATH:**

Твой `uv.exe` находится здесь:
```
C:\Users\Pavlo\AppData\Local\Programs\Python\Python313\Scripts\uv.exe
```

Добавь эту папку в системный PATH:
1. Win + R → `sysdm.cpl` → Дополнительно → Переменные среды
2. В разделе "Переменные пользователя" найди `Path` → Изменить
3. Добавь строку: `C:\Users\Pavlo\AppData\Local\Programs\Python\Python313\Scripts`
4. OK → OK → перезапусти терминал

**Проверка:**
```cmd
uv --version
```
Должно вывести что-то вроде `uv 0.x.x`.

## Первоначальная настройка

### Шаг 1 — Создай `.env` файл

В папке проекта (`mechabellum_replay_parser/`) создай файл `.env`:

```env
# Обязательно
OPENAI_API_KEY=sk-...твой_ключ...
PLAYER_NAME=ТвоёИмяВИгре

# Папка с реплеями Steam (путь на твоём компьютере)
MECHABELLUM_REPLAY_DIR=C:\Program Files (x86)\Steam\steamapps\common\Mechabellum\ProjectDatas\Replay

# Необязательно — настройки по умолчанию и так хорошие
OPENAI_MODEL=gpt-4o
LLM_TIMEOUT_SECONDS=60
REPLAY_AFTER_PROCESS=delete
```

> **PLAYER_NAME** — точное написание ника как в игре (важно!). Тренер ищет твои данные по имени.

### Шаг 2 — Установи зависимости

```bash
cd mechabellum_replay_parser
uv sync
```

### Шаг 3 — Запусти Docker

```bash
docker-compose up -d
```

Это поднимает:
- `core-api` — FastAPI сервер на порту 8000 (парсер + AI + вотчер)
- `postgres` — база данных для хранения истории
- `redis` — очередь событий (зарезервирован для будущего)

Подождать первого старта ~30 секунд. Проверить что всё работает:
```bash
curl http://localhost:8000/health
# Должно вернуть: {"status": "ok"}
```

### Шаг 4 — Примени миграции базы данных

```bash
uv run alembic upgrade head
```

Это создаёт таблицы в PostgreSQL для хранения истории партий и рекомендаций.

### Шаг 5 — Загрузи базу знаний

```bash
uv run mechabellum knowledge ingest
```

Это разбивает `game_knowledge.md` на 95 смысловых кусков — тренер будет брать только нужные разделы для каждой ситуации.

Делается один раз. Повтори если обновил `game_knowledge.md`.

Тебе нужно запустить **два процесса**:

### Терминал 1 — Docker-бэкенд

```bash
docker-compose up
```

Оба процесса держи запущенными пока играешь.

## Пользовательский флоу во время игры

### Каждый раунд выглядит так:

```
Ты нажал "Готов" в Mechabellum
         │
         ▼
[Окно] Ввод supply
  ┌─────────────────────────────────┐
  │  РАУНД 4                        │
  │                                 │
  │  Твой текущий supply:  [____]   │
  │                                 │
  │   [Подтвердить]  [Пропустить]   │
  └─────────────────────────────────┘
         │  (вводишь например 350)
         ▼
[Окно] AI анализирует позицию...
  ┌─────────────────────────────────┐
  │  ●  ●  ●                        │
  │  AI анализирует позицию...      │
  └─────────────────────────────────┘
         │  (несколько секунд)
         ▼
[Окно] Результат
  ┌─────────────────────────────────┐
  │  ⚔ Раунд 4 · ТвоёИмя          │
  │─────────────────────────────────│
  │  Купи Arclight, поставь в центр │
  │─────────────────────────────────│
  │  У противника 3 Wasp — воздушная│
  │  угроза высокая. Arclight закрое│
  │  т воздух и поможет против      │
  │  Crawler. Поставь ближе к центру│
  │  чтобы прикрыть фланги...       │
  │─────────────────────────────────│
  │  РАССТАНОВКА                    │
  │  [  визуальная доска юнитов  ]  │
  │─────────────────────────────────│
  │      [👍 Помогло]  [👎 Нет]    │
  └─────────────────────────────────┘
```

### Что делать с советом

1. **Читаешь жирный текст вверху** — это главный совет (что купить / главная идея)
2. **Читаешь объяснение** — почему именно так, на что обратить внимание
3. **Смотришь на доску** — зелёные круги это куда ставить юниты, серые — где они сейчас
4. **Делаешь ходы в Mechabellum** — покупаешь и расставляешь согласно совету (или нет — ты главный)
5. **Жмёшь 👍 или 👎** — необязательно, но помогает собирать статистику

### Легенда доски

| Обозначение | Значение |
|---|---|
| 🟢 Зелёный круг | Рекомендуемая позиция (новый или перемещённый юнит) |
| ⚫ Серый круг | Текущая позиция юнита из реплея |
| 🟨 Золотой квадрат | Постройка (Supply Tower / Command Tower / Research Tower) |

### Если пропустить ввод supply

Нажми **"Пропустить"** — тренер всё равно проанализирует позицию, просто не будет учитывать бюджет на покупки.

### Если окно зависло на "Загрузка"

OpenAI отвечает медленно. По умолчанию тайм-аут 60 секунд, после которого тренер выдаёт запасной план (держать текущие позиции).

## CLI-команды для анализа прошлых реплеев

Эти команды работают без Docker, напрямую с `.grbr` файлами.

### Посмотреть историю партии

```bash
uv run mechabellum-replay-parser battle "путь\к\реплею.grbr"
```

Выводит таблицу: каждый раунд, что купил каждый игрок, результат боя.

### Посмотреть техи

```bash
uv run mechabellum-replay-parser tech "путь\к\реплею.grbr"
```

Список всех изученных техов на каждого игрока.

## Режим отладки (DEBUG)

Добавь в `.env`:

```env
DEBUG=true
```

В этом режиме:
- Тренер **не вызывает AI**, только парсит реплей
- Пишет файлы в папку `.debug/` — полезно чтобы понять что именно было проанализировано
- Файлы: `latest_parsed.json`, `latest_state_view.json`, `latest_planner_response.json` и т.д.

Убери `DEBUG=true` чтобы вернуть нормальную работу.

### Режим без базы данных

```env
DEBUG_NO_DB=true
```

Тренер работает в полную силу (AI советы работают), но ничего не сохраняет в PostgreSQL. Удобно если не хочешь запускать Docker.

## Быстрый старт без Docker

Если хочешь запустить **без Docker** (только для разработки или тестирования):

```bash
# Терминал 1 — API сервер напрямую
DEBUG_NO_DB=true uv run uvicorn mechabellum_replay_parser.api.app:app --reload

# Терминал 2 — нативное окно
uv run mech-native-ui
```

Советы будут работать, база данных — нет.

## Структура данных которые сохраняются

После каждой партии в базе сохраняется:

| Что | Зачем |
|---|---|
| Файл реплея, раунд, игрок | Идентификация партии |

## Возможные проблемы

### Окно не появляется

- Убедись что запущен `mech-native-ui` в отдельном терминале
- Проверь что `docker-compose up` поднял `core-api` (`curl http://localhost:8000/health`)

### "Ожидаю новую партию" не меняется

- Steam должен сохранять реплей в конце каждого раунда подготовки
- Убедись что путь `MECHABELLUM_REPLAY_DIR` в `.env` совпадает с реальной папкой реплеев
- Стандартный путь: `C:\Program Files (x86)\Steam\steamapps\common\Mechabellum\ProjectDatas\Replay`

### AI отвечает медленно или не отвечает

- Проверь `OPENAI_API_KEY` в `.env`
- Увеличь тайм-аут: `LLM_TIMEOUT_SECONDS=120`
- После тайм-аута тренер выдаёт запасной план (держать позиции)

### Ошибка с базой данных при старте

- Это не критично — тренер продолжит работу без сохранения
- Чтобы убрать ошибку: `docker-compose up postgres` (только база данных)
- Или добавь `DEBUG_NO_DB=true` в `.env` чтобы полностью отключить БД

### Неправильное имя игрока

Тренер не находит твои данные в реплее. Убедись что `PLAYER_NAME` в `.env` написано **точно так же** как в игре — с теми же заглавными/строчными буквами.

## Типичная сессия шаг за шагом

```
1. Открываешь Mechabellum
2. cd mechabellum_replay_parser
3. docker-compose up -d
4. uv run mech-native-ui        ← держишь открытым рядом с игрой

5. Играешь раунд → жмёшь Готов
6. В окне тренера: вводишь supply → Подтвердить
7. Ждёшь 5-15 секунд
8. Читаешь совет, смотришь доску
9. Делаешь ходы в игре
10. Повторяешь с шага 5 каждый раунд

11. После сессии: закрываешь окно тренера
12. docker-compose down   (или оставляешь запущенным)
```
